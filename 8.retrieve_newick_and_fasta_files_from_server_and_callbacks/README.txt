
This example utilizes the fasta file and the newick file.

The strains are shown in the order they are in the newick file.

This sample only works when served from a web server.
It uses web protocols to read the fasta file.

This also contains Javascript functions defined for call backs.
This gives the developer more control over what is displayed on the page when there is an error
and the ability to trigger other code when the Snip Viz has finished loading.
